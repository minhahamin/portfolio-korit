실무에서 마주친 문제와 해결 과정입니다.

MES 생산관리 시스템

LOT 번호 자동 생성 중복 이슈

Problem

여러 사용자가 동시에 LOT 번호를 생성할 때 중복 번호가 발행되는 문제 발생

Cause

MAX(lot_no) + 1 방식으로 생성하던 로직에서 동시 요청 시 같은 값을 읽어 Race Condition 발생

Solution

트랜잭션 격리 수준 조정 및 DB 시퀀스 기반 자동증가 방식으로 변경, UNIQUE INDEX 추가 후 중복 시 재시도 로직 구현

Result

LOT 번호 중복 발생 0건, 고부하 동시 요청 환경에서도 안정적인 번호 생성 보장

Node.jsMariaDBTransactionIndex

현장 안전관리 시스템

JWT 토큰 만료 후 API 요청 처리

Problem

액세스 토큰 만료 시 사용자가 작업 중 강제 로그아웃되는 UX 문제

Cause

토큰 만료 시 재발급 로직 없이 401 에러만 반환하는 단순한 인증 구조

Solution

Axios 인터셉터에서 401 응답 감지 시 리프레시 토큰으로 자동 재발급 처리 후 원래 요청 재시도 구현

Result

토큰 만료에 따른 불필요한 로그아웃 완전 제거, 사용자 작업 중단 없이 세션 유지

MES 생산관리 시스템

대용량 데이터 조회 최적화

Problem

생산 이력 조회 화면에서 5,000건 이상 데이터 조회 시 응답 시간 5초 이상 소요

Cause

인덱스 미설정 컬럼(생산일자, LOT번호) 조건 조회로 Full Table Scan 및 N+1 쿼리 발생, 페이징 미구현

Solution

조회 조건 컬럼에 복합 인덱스 추가, JOIN 기반 쿼리로 N+1 제거, 서버 페이징 처리 및 프론트 가상 스크롤 적용

Result

평균 응답 시간 5,200ms → 280ms (약 95% 개선)

Node.jsMariaDBSQL OptimizationReact

MES 생산관리 시스템

API 병렬 호출로 대시보드 성능 개선

Problem

대시보드 초기 로딩 시 여러 API를 순차 호출하여 3~4초의 로딩 시간 발생

Cause

독립적인 API(생산 현황, KPI, 재고, 공지사항)를 순차적으로 호출하는 구조로 불필요한 대기 시간 누적

Solution

Promise.all을 활용해 독립 API를 병렬 호출로 변경, 변경이 적은 기준 정보 클라이언트 캐싱 적용

Result

대시보드 초기 로딩 3,500ms → 800ms (약 77% 개선), 사용자 체감 속도 크게 향상
